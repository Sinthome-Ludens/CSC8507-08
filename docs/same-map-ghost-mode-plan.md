# 同图竞速联机模式计划文档

## 目标概述
新增第二种联机模式“同图竞速 + 幽灵对手”。

该模式与现有“异图竞速”共存，但首版先使用隐藏配置切换，不改大厅完整 UI。模式开启后：

- 三张地图序列只由服务端生成
- 客户端接收并使用与服务端完全一致的三张地图
- 双方在同一张地图上各自进行竞速
- 对手以“幽灵”形式显示在本地地图中
- 幽灵只做视觉展示，不参与玩法判定

## 功能范围

### 首版纳入范围
- 新增联机模式枚举，支持旧模式与新模式共存
- 服务端权威生成三张地图序列并同步给客户端
- 客户端按服务端下发序列进入和推进地图
- 支持跨关、Retry 时保持或重发同图序列
- 在本地场景中显示远端玩家幽灵
- 幽灵支持位置与朝向同步、插值和平滑显示
- 保持现有三关竞速结算逻辑可用

### 首版不纳入范围
- 不做完整大厅模式选择 UI
- 不支持主机手动选图
- 不同步完整角色动作状态
- 不实现共享交互、共享敌人或合作玩法
- 不让幽灵参与碰撞、伤害、道具、门、终点或敌人逻辑

## 核心设计

### 联机模式扩展
需要新增联机比赛模式枚举，例如：

- `DifferentMapRace`
- `SameMapGhostRace`

该模式存放在 `Res_Network` / `Res_LobbyState` 中，首版通过隐藏配置使用。

### 地图序列权威化
当前地图序列由本地生成。新模式下改为：

- 仅服务端生成三张随机地图
- 客户端不再本地生成同图模式地图序列
- 服务端通过可靠包下发：
  - 当前联机模式
  - `mapSequence[3]`
  - 当前 `roundIndex`
- 客户端收到后才允许进入首关或继续切关

### 同图推进规则
同图模式仍沿用三关竞速框架：

- 双方进入同一张地图
- 各自在本地完成自己的关卡流程
- 服务端继续权威结算胜负
- `NextLevel` 按统一序列推进
- `Retry` 时由服务端重新生成并广播新的三图序列

### 幽灵实体语义
远端玩家不直接作为本地真实玩家实体存在，而是使用独立幽灵实体。

幽灵实体应具备：

- 可见模型
- `Transform`
- 插值缓冲
- 与本地玩家明显区分的材质表现

幽灵实体不应具备：

- `C_T_Player`
- 本地输入组件
- 玩家状态组件
- 可参与物理的碰撞/刚体逻辑
- 可被敌人、门、道具、终点系统识别为玩家的标签

## 实现方案

### 协议层
在 `Net_Protocol.h` 中扩展协议：

- `SYNC_MULTIPLAYER_SETUP`
- `SYNC_GHOST_TRANSFORM`
- `CLIENT_GHOST_TRANSFORM`

用于承载：

- 联机模式
- 三张地图序列
- 当前轮次
- 远端玩家位置与旋转

### 运行态资源
需要扩展运行态状态，至少包括：

- 当前联机模式
- 当前权威地图序列
- 客户端是否已收到序列
- 本地玩家实体 ID
- 当前场景远端幽灵实体 ID

### 地图进入与切场景
同图模式进入联机 bootstrap 场景：

- Host 和 Client 先建立会话
- 服务端下发权威地图序列
- 双方收到后统一切入第一张地图
- 跨场景保留联机会话时，不丢失模式与地图序列

### 幽灵同步与渲染
首版复用现有插值框架，但不让幽灵混入本地真实玩家逻辑。

推荐流程：

- 本地玩家继续使用本地玩法实体
- 额外同步本地玩家位姿到远端
- 对端创建纯视觉幽灵实体
- 幽灵实体通过 `C_D_InterpBuffer` 做平滑插值

可视化要求：

- 半透明或固定高亮色
- 与本地玩家明显区分
- 平滑移动，避免抖动

### 系统隔离
需要确保以下系统不会误处理幽灵：

- `Sys_EnemyVision`
- `Sys_EnemyAI`
- `Sys_Door`
- `Sys_Item`
- `Sys_LevelGoal`
- `Sys_DeathJudgment`
- 与本地玩家标签直接相关的其他系统

## 测试计划

### 功能测试
- Host 开启同图模式，Client 连接后双方进入相同首图
- 连续完成第 1、2 关时，双方进入完全相同的下一张图
- Retry 后双方进入同一组新地图，不沿用旧序列
- Client 断线重连或异常切换后不会错误回退到本地随机序列

### 幽灵测试
- 双方都能看到对手幽灵
- 幽灵位置和朝向连续平滑
- 幽灵不会阻挡本地移动
- 幽灵不会触发敌人感知
- 幽灵不会触发门、道具、终点、死亡判定

### 回归测试
- 旧“异图竞速”模式保持现有行为
- 原有比赛阶段推进与 GameOver 结算不回归
- 跨关保活网络会话不丢失模式状态
- 现有 UI/HUD 的进度与结果展示继续可用

## 默认决策与假设
- 第二种模式与旧模式长期共存
- 首版不做大厅 UI，只做隐藏配置切换
- 选图仍沿用当前“随机三张”逻辑，但由服务端独占生成
- 幽灵首版为纯视觉实体，只同步位姿，不同步完整动作状态
- 同图模式仍是竞速，不扩展为合作玩法
